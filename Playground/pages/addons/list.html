LLM Guide - nui-list (read this first)

List Component

High-performance virtualized list with search, sort, and selection. Handles 50,000+ items smoothly through element reuse and virtual scrolling. Theoretical max: ~300K items (at 60px height; actual height is auto-detected from the first rendered item).

Overview

The <nui-list> component provides a performant, feature-rich list with:

  • Virtual Scrolling - Only renders visible items (~20-30) regardless of dataset size
  • Element Reuse - DOM elements created once and reused for performance
  • Lazy Loading - Deferred content loading via update() callback when items enter DOM; works for images, API calls, or any expensive operation
  • Search & Filter - Multi-property search with debounced input
  • Sort - Multi-column sorting with direction toggle
  • Selection - Single or multi-select with Ctrl/Shift support
  • Two Modes - Normal (<1000 items) and Fixed (≥1000 items) for optimal performance

Import Requirements

Layout Constraints (Important)

Virtual Scroller Boundaries: Because <nui-list> is a virtual scroller, it needs to know what is currently visible on the screen. It MUST be placed inside a container with a determinable height.

If you place the list in a container that dictates its height by its children (e.g., standard document flow, or <main overflow-y="auto">), the list will think its viewport is infinite, breaking virtual memory and potentially returning 0 visual space.

Rules:

  • Provide an explicit height constraint on the list's parent. The container must not rely on intrinsic sizing (growing based on its contents). Instead, it must have a determinable, bounded dimension (e.g., via flex-grow, height: 100%, or Grid layouts) so the virtual scroller can accurately measure its viewport.
  • All items must be the same height. The virtual scroller auto-detects height from the first rendered item and uses that for all calculations. If items vary in height, visibility detection will be incorrect and items may be skipped or duplicated during scroll. Use the Lazy Loading pattern with update_delay to defer rendering of complex content until after the item is positioned.
  • If using flexbox, make sure the parental container has fixed bounds and the list wrapper has flex: 1; overflow: hidden;.
  • If inserting into a <nui-dialog mode="page">, you must add content-scroll="false" to the dialog to disable native HTML scroll physics and permit the list to own the bounds!
  • Visibility Contexts (e.g. Dialogs): The virtual scroller measures layout height. If housed in a container that uses display: none (like a hidden <dialog> or tab), it cannot calculate how many rows to render and will paint a blank view. Be sure to call list.loadData(...) AFTER the display state becomes visible (e.g., after calling dialog.showModal()) so the list measures its bounds correctly!

Lazy Loading

The list supports deferred content loading through an update() callback. This pattern fires when items are rendered to the DOM, enabling on-demand loading for any expensive operation—not just images. Use it for API calls, complex DOM generation, or any content that shouldn't be created until needed.

How it works: Each rendered element can have an update() method that's called when the element is appended to the DOM. Add update_delay to defer execution. This is a general pattern—use it for images, API calls, map initialization, or any heavy lifting.

API Reference

loadData(options)

Initializes or updates the list with data and configuration. Call this on an <nui-list> element.

Configuration Options

Option Type Description
data array Array of data items to display in the list.
render function(item) Function that receives a data item and returns an HTMLElement to render it. See Lazy Loading section for the optional update() method pattern.
search array Properties to search. Each entry is { prop: 'fieldName' }. Adds a search input to the header.
sort array Sort options. Each entry is { label: 'Display Name', prop: 'fieldName', numeric?: boolean }. Adds a sort dropdown and direction toggle to the header.
sort_default number Index of the default sort option (default: 0).
sort_direction_default 'up' | 'down' Default sort direction (default: 'up' = ascending).
filters array Filter dropdowns to add to the header. Each entry: { prop: 'fieldName', label?: 'Label', options: [{ value: 'val', label: 'Label' }], default?: 'val' }
footer object Footer configuration with buttons_left and/or buttons_right arrays. Each button: { label: 'Text', type: 'primary|danger|outline', fnc: () => {} }
events function(event) Callback for list events. See Events below.
logmode boolean When true, list auto-scrolls to bottom when new items are appended. Useful for log/chat interfaces.
single boolean When true, only one item can be selected at a time (no multi-select with Ctrl/Shift).
verbose boolean When true, enables debug console logging.

Component Methods

Method Returns Description
loadData(options) void Initialize the list with data and options.
update(force?) void Re-renders visible items. Use update(true) to force a full re-render.
updateData(data, skipFilter?) void Replace the data array. Set skipFilter=true to skip re-applying current filters.
appendData() void Append new items that were pushed to the original data array. Use with logmode.
getSelection(full?) array Get selected items. If full=true, returns full data objects; otherwise returns original indices.
getSelectedListIndex() array Returns array of selected indices in the filtered/sorted list (not original indices).
setSelection(indexArray) void Set selected items by original indices. Pass empty array to clear selection.
updateOptions(options) void Update options after initialization. Can add/replace search, sort, or filters.
updateItem(idx, data?, force?) void Update a single item by original index. Optionally provide new data and force re-render.
updateItems(items) void Batch update multiple items. Each entry is { idx: number, data?: any } or just a number.
scrollToIndex(index) void Scroll to bring the item at the given filtered list index into view.
reset() void Reset scroll position and re-render current view.
cleanUp() void Remove the list, clean up event listeners and observers. Call before removing from DOM.

Events

The events callback receives an event object with type, value, and other properties:

Event Type Properties Description
'selection' value (count) Fired when selection changes. Use getSelection() to get actual items.
'sort' index, direction Fired when sort column or direction changes.
'filter' prop, value Fired when a filter dropdown value changes.
'search_input' value Fired when search input changes (debounced 100ms).
'visibility' value (true/false) Fired when list enters or leaves the viewport.
'height_change' value (height) Fired when an item's height changes (for virtual scroll recalculation).
'list' value ('filtered' | 'reset') General list events.
