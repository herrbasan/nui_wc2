LLM Guide - nui-list

List Component

High-performance virtualized list with integrated search, sort, filter, and selection. Handles thousands of items by rendering only what's visible.

Full-Featured Demo

A complete example with 5,000 items, images, search, multi-column sort, category filter, and footer actions.

Anatomy of a List

To build a list like the demo above, you need three things: data, a render function, and container layout.

1. The Container

The list positions itself absolutely to fill its container. The container needs position: relative and an explicit height.

2. The Data

Any array of objects. The list adds internal properties (oidx for original index) but your data structure is preserved.

3. The Render Function

A function that receives a data item and returns an HTMLElement. This is called only for visible items.

4. Configuration

Pass everything to loadData(). The list builds the header (search, sort, filters) and footer automatically based on what you provide.

Variants and Modes

Minimal List (No Header, No Footer)

Just data and a render function. No search, sort, or filters.

Log Mode

API Reference

loadData(options)

Initializes the list with data and configuration.

OptionTypeDescription
dataarrayRequired. Array of data objects.
renderfunction(item)Required. Returns HTMLElement for each item.
searcharrayProperties to search: [{ prop: 'name' }]
sortarraySort options: [{ label: 'Name', prop: 'name', numeric: true }]
filtersarrayFilter dropdowns with options array.
footerobject{ buttons_left: [], buttons_right: [] }
selection'multi'|'single'Enable selection (default: 'single').
logmodebooleanAuto-scroll to bottom for logs.
eventsfunction(e)Callback for all list events.
