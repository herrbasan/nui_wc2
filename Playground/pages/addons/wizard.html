Strategy 1: Native HTML5 Validation

In mode="required", the wizard automatically validates any <form> inside the current step using reportValidity() before allowing navigation. No custom code needed.

Strategy 3: Async Server-Side Validation

Assign a Promise to e.detail.promise for server-side checks. The wizard automatically shows a busy state and advances when the promise resolves. Rejections show an error status.

Strategy 4: Manual Status Control

Set step status directly from any event handler. This is useful for real-time validation feedback as the user types, without waiting for Next to be clicked.
