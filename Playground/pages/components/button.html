Button Component

Enhanced button elements with consistent styling and interaction patterns

Overview

The <nui-button> component wraps standard HTML <button> elements with enhanced styling and optional features. It maintains semantic HTML while providing consistent visual design and interaction patterns across the application.

Basic Usage

Wrap a standard button element:

<nui-button> <button type="button">Click Me</button> </nui-button>

Button Types

Use the type attribute for different button behaviors:

<!-- Submit button (form submission) --> <nui-button> <button type="submit">Submit Form</button> </nui-button> <!-- Reset button (clear form) --> <nui-button> <button type="reset">Reset</button> </nui-button> <!-- Regular button (default) --> <nui-button> <button type="button">Action</button> </nui-button>

Buttons with Icons

Combine buttons with the <nui-icon> component:

<nui-button> <button type="button"> <nui-icon name="save"></nui-icon> Save </button> </nui-button> <nui-button> <button type="button"> Delete <nui-icon name="delete"></nui-icon> </button> </nui-button> <!-- Icon-only button --> <nui-button> <button type="button" aria-label="Settings"> <nui-icon name="settings"></nui-icon> </button> </nui-button>

Disabled State

Use the standard disabled attribute:

<nui-button> <button type="button" disabled>Disabled Button</button> </nui-button>

Event Handling

Use standard event listeners or the NUI action system:

// Standard JavaScript const button = document.querySelector('button'); button.addEventListener('click', () => { console.log('Button clicked!'); }); // NUI action system (attribute-based) // See action system documentation <!-- Using nui-event-* attributes --> <nui-button> <button type="button" nui-event-click="show-alert"> Show Alert </button> </nui-button>

Styling

Button appearance is controlled via CSS. Customize using CSS custom properties or by targeting the button element:

/* Custom button styling */ nui-button button { padding: 0.75rem 1.5rem; border-radius: 4px; background: var(--nui-accent); color: var(--nui-text-on-accent); border: none; cursor: pointer; font-weight: 600; } nui-button button:hover { background: var(--nui-accent-hover); } nui-button button:disabled { opacity: 0.5; cursor: not-allowed; }

Accessibility

The button component maintains full accessibility by using semantic HTML:

  • Semantic Element: Uses standard <button> element with proper type attribute
  • Keyboard Navigation: Native focus and activation with Tab and Enter/Space keys
  • Screen Reader Support: Button role and label automatically recognized
  • Icon-Only Buttons: Use aria-label for buttons with only icons
  • Disabled State: disabled attribute properly communicated to assistive technology
<!-- Good: Icon-only button with aria-label --> <nui-button> <button type="button" aria-label="Close dialog"> <nui-icon name="close"></nui-icon> </button> </nui-button> <!-- Good: Button with visible text --> <nui-button> <button type="button"> <nui-icon name="save"></nui-icon> Save Changes </button> </nui-button>

Best Practices

  • Always specify type attribute (default is "submit" which may cause unwanted form submission)
  • Use descriptive button text that clearly indicates the action
  • Provide aria-label for icon-only buttons
  • Use disabled attribute for unavailable actions, not CSS pointer-events
  • Consider button placement and visual hierarchy in the UI
  • Test keyboard navigation and focus states
