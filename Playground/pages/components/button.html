Button Component

Enhanced button elements with consistent styling and interaction patterns

Overview

The <nui-button> component wraps standard HTML <button> elements with enhanced styling and optional features. It maintains semantic HTML while providing consistent visual design and interaction patterns across the application.

Basic Usage

Wrap a standard button element:

<nui-button>
  <button type="button">Click Me</button>
</nui-button>

Button Types

Use the type attribute for different button behaviors:

<!-- Submit button (form submission) -->
<nui-button>
  <button type="submit">Submit Form</button>
</nui-button>

<!-- Reset button (clear form) -->
<nui-button>
  <button type="reset">Reset</button>
</nui-button>

<!-- Regular button (default) -->
<nui-button>
  <button type="button">Action</button>
</nui-button>

Buttons with Icons

Combine buttons with the <nui-icon> component:

<nui-button>
  <button type="button">
    <nui-icon name="save"></nui-icon>
    Save
  </button>
</nui-button>

<nui-button>
  <button type="button">
    Delete
    <nui-icon name="delete"></nui-icon>
  </button>
</nui-button>


<nui-button>
  <button type="button" aria-label="Settings">
    <nui-icon name="settings"></nui-icon>
  </button>
</nui-button>

Disabled State

Use the standard disabled attribute:

<nui-button>
  <button type="button" disabled>Disabled Button</button>
</nui-button>

Button Container

The <nui-button-container> component manages the layout and alignment of a group of buttons.

Attributes

  • align: Controls horizontal alignment. Options: start (default), center, end, between.
  • gap: Controls spacing between buttons. Keywords: none, small, medium, large. Also accepts custom values like 2rem or CSS variables (e.g., var(--nui-space)). Default is small (var(--nui-space-half)).
  • direction: Layout direction. Options: row (default), column.

Examples

Right Aligned:

<nui-button-container align="end">
  <nui-button><button type="button">Cancel</button></nui-button>
  <nui-button><button type="button">Confirm</button></nui-button>
</nui-button-container>

Centered with Large Gap:

<nui-button-container align="center" gap="large">
  <nui-button><button type="button">Option A</button></nui-button>
  <nui-button><button type="button">Option B</button></nui-button>
</nui-button-container>

Column Layout:

<nui-button-container direction="column" align="stretch">
  <nui-button><button type="button" style="width: 100%">Top</button></nui-button>
  <nui-button><button type="button" style="width: 100%">Bottom</button></nui-button>
</nui-button-container>

Interactive Examples

Toggle Theme

Toggle between light and dark mode using the built-in toggle-theme action:

<nui-button nui-event-click="toggle-theme">
  <button type="button">Toggle Theme</button>
</nui-button>

Toggle Class

Toggle a CSS class on a target element using toggle-class:className@selector:

<!-- Target element -->
<div id="demo-box" class="demo-box">Target Box</div>

<!-- Button toggles 'highlight' class on #demo-box -->
<nui-button nui-event-click="toggle-class:highlight@#demo-box">
  <button type="button">Toggle Highlight</button>
</nui-button>

Custom Actions

You can also register custom actions in JavaScript:

// Register action
nui.registerAction('show-alert', (target, source) => {
  alert(`Clicked: ${source.textContent.trim()}`);
});
 
<nui-button nui-event-click="show-alert">
  <button type="button">Show Alert</button>
</nui-button>

Styling

Button appearance is controlled via CSS. Customize using CSS custom properties or by targeting the button element:

/* Custom button styling */
nui-button button {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  gap: 0.5rem;
  padding: 0.75rem 1.5rem;
  border-radius: 4px;
  background: var(--nui-accent);
  color: var(--nui-text-on-accent);
  border: none;
  cursor: pointer;
  font-weight: 600;
}

nui-button button:hover {
  background: var(--nui-accent-hover);
}

nui-button button:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}

Accessibility

The button component maintains full accessibility by using semantic HTML:

  • Semantic Element: Uses standard <button> element with proper type attribute.
  • Keyboard Navigation: Native focus and activation with Tab and Enter/Space keys.
  • Screen Reader Support: Button role and label automatically recognized.
  • Icon-Only Buttons: Use aria-label for buttons with only icons.
  • Disabled State: disabled attribute properly communicated to assistive technology.
<!-- Good: Icon-only button with aria-label -->
<nui-button>
  <button type="button" aria-label="Close dialog">
    <nui-icon name="close"></nui-icon>
  </button>
</nui-button>

<!-- Good: Button with visible text -->
<nui-button>
  <button type="button">
    <nui-icon name="save"></nui-icon>
    Save Changes
  </button>
</nui-button>

Best Practices

  • Always specify type attribute (default is "submit" which may cause unwanted form submission).
  • Use descriptive button text that clearly indicates the action.
  • Provide aria-label for icon-only buttons.
  • Use disabled attribute for unavailable actions, not CSS pointer-events.
  • Consider button placement and visual hierarchy in the UI.
  • Test keyboard navigation and focus states.
