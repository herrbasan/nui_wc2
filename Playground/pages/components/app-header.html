LLM Guide - nui-app-header

App Header (<nui-app-header>)

A standardized application top bar with built-in layout management and semantic zones.

Overview

The <nui-app-header> acts as a semantic flex-container, replacing complex monolithic top-bar code with a standard structure. It divides the space into three zones using the slot attribute (or class names for compatibility): left, center, and right.

When placed inside an <nui-app> shell, it automatically docks to the top and works in tandem with <nui-sidebar> to manage screen real estate and keyboard accessibility zones.

Status Badges

Any element inside the header can accept a data-badge attribute to display an overlaid indicator or notification count. This removes the need for extra markup wrapping.

An empty string data-badge="" creates a minimal dot indicator, while text or numbers create pill-shaped badges.

Interactive Delegation (data-action)

The app shell relies heavily on standard nui-action events to communicate between the Header and the Sidebars without custom JavaScript wiring.

  • data-action="toggle-sidebar" - Toggles the left sidebar.
  • data-action="toggle-sidebar:right" - Toggles the right sidebar.

You can see this in action by clicking the menu button or settings button in the main header of the Playground above.
