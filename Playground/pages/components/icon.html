LLM Guide - nui-icon

Icon Component

SVG sprite-based icon system for consistent, scalable icons

Overview

The <nui-icon> component uses an SVG sprite system for efficient icon rendering. All icons are loaded from a single sprite file (material-icons-sprite.svg) which reduces HTTP requests and allows for instant icon rendering.

Features

  • SVG Sprite System - Single file contains all icons, efficient loading
  • Scalable - Vector-based, scales perfectly at any size
  • CSS Themeable - Icons inherit color from CSS, easy to customize
  • Material Design Icons - Comprehensive set of modern, clean icons
  • Inline Usage - Can be used inline with text

Implementation Details

SVG Sprite System

All icons are stored in a single SVG sprite file with symbol definitions:

<svg xmlns="http://www.w3.org/2000/svg">
	<symbol id="icon-search" viewBox="0 0 24 24">
		<path d="M15.5 14h-.79l-.28-.27A6.471..."/>
	</symbol>
</svg>

Benefits

  • Single HTTP Request - All icons loaded once, cached by browser
  • No Font Files - True vector graphics, not font characters
  • Instant Rendering - No FOUC (Flash of Unstyled Content)
  • Easy Updates - Regenerate sprite to add/update icons
  • CSS Control - Full styling control via CSS

Sprite Generation

The sprite is generated from individual SVG files using generate_icon_sprite.py. This allows easy addition of new icons - just add SVG files and regenerate the sprite.
