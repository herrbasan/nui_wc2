LLM Guide - Accessibility

Accessibility

NUI is built with a "Performance & Accessibility First" mindset, ensuring high-quality experiences for everyone via intelligent platform integration.

Philosophy: DOM-First Accessibility

NUI follows a DOM-first accessibility approach with intelligent context detection. Instead of complex framework abstractions, we lean on the browser's native capabilities and enhance them with custom behaviors where necessary.

Core Principle: Enhance, don't replace. We use native elements like <button>, <dialog>, and <input> as the foundation, ensuring they work even if JavaScript fails.

Intelligent Accessibility Upgrades

One of NUI's most powerful features is its ability to automatically detect and fix common accessibility gaps during initialization.

Smart Label Generation

Icon-only buttons automatically receive aria-label attributes based on their icon names and surrounding context.

Keyboard Interactivity

NUI implements a Hybrid Navigation Model that balances standard web expectations with application-like efficiency.

Universal Keys
  • Tab: Navigate between interactive elements.
  • Enter / Space: Activate buttons, links, and toggles.
  • Escape: Close dialogs, overlays, and menus.
  • Arrows: Navigate within lists, trees, and tab bars.
  • Home / End: Jump to start or end of collections.

Focus Management

NUI handles the complex logic of managing focus so you don't have to.

Focus Traps & Roving Tabindex

Dialogs automatically trap focus within their bounds. Composite components like nui-tabs and nui-link-list implement roving tabindices to reduce the number of tab stops while remaining navigable via arrows.

Component Support

Every NUI component is audited for ARIA compliance and keyboard support.

nui-sortable

Implements Keyboard Reordering. Users can "grab" items with Space, move them with Arrows, and "drop" them to reorder. Every move is announced to screen readers.

nui-tabs

Follows the ARIA Tab Pattern. Uses role="tablist", role="tab", and role="tabpanel". Supports arrow key switching and automatic focus state synchronization.

nui-select

Transforms from a Combobox on desktop to an accessible Mobile Bottom Sheet. Includes auto-focus for search for immediate filtering.

nui-table

Uses a Responsive Data-Label Pattern. On small screens, table rows transform into cards, but data-label attributes ensure screen readers maintain column context.

nui-dialog

Built on the native <dialog> element. Handles backdrop clicks, Escape key dismissal, and returns focus to the trigger on close.

nui-progress

Automatically maps role="progressbar" and standard ARIA range attributes (min/max/now) for accurate screen reader feedback.

Developer Utilities

NUI provides tools to help you build accessible apps and debug potential issues.

a11y.announce()

A hidden live-region utility for announcing dynamic state changes that aren't inherently visible (e.g., "Item deleted" or "Sort order changed").

a11y.warn()

NUI audits your markup in real-time. If it finds a component missing critical accessibility data (like a label), it will fix it automatically and log a teaching warning in the console.

Visual & Motion Preferences

  • Reduced Motion: NUI respects prefers-reduced-motion, disabling or shortening animations automatically.
  • High Contrast: Theme tokens use adaptive colors (light-dark()) and maintain WCAG AA contrast ratios (4.5:1 for text).
